# Analysis: sample_tclk.tcl

## What It Does

This is a CGI script invoked directly by a web server (originally fronted by a Perl auth wrapper that was retired in 2005; no replacement). It reads named parameters from `QUERY_STRING`, decodes each through a `url_decode` that invokes `subst` on the decoded text, and dispatches on the `action` parameter to one of fifteen branches. Each branch executes an operator-level operation against the host: shelling out to `ping`, `traceroute`, `tail`, `grep`, `psql`, `telnet`, `curl`, `dmesg`, or `/etc/init.d/*`; reading log files under `/var/log/`; sourcing Tcl plugins; rendering operator templates through `subst`; or directly `eval`-ing a user-supplied Tcl string.

Hidden control flow:
- All branches use plain `if` rather than `elseif`/`switch`, so multiple actions can fall through and execute in the same request if the parameters are crafted to match more than one (only one `action` matches at a time, but every branch is re-evaluated unconditionally).
- `url_decode` runs `subst` on every decoded parameter before any branch sees it, so command substitution fires on `host`, `cmd`, `filter`, and `pattern` regardless of which `action` is requested.
- `exec` calls and `source` lack `catch`, so failures terminate the script mid-response after headers and partial HTML have been written.
- Output is written without HTML escaping, and Content-Type is the only header sent â€” no `X-Frame-Options`, no `Content-Security-Policy`, no auth challenge.

## Risk Flags

- **[Critical]** `url_decode` calls `subst $str` after percent-decoding â€” every parameter passed through it (`host`, `cmd`, `filter`, `pattern`) is run through Tcl command substitution. A query string containing `%5bexec%20id%5d` (`[exec id]`) executes `id` before any action branch runs. This is unauthenticated remote code execution on every request that supplies any of those parameters.
- **[Critical]** `action=tcl` branch runs `eval $cmd` with no authentication, no allowlist, no safe interpreter. Arbitrary Tcl execution as the CGI user. The comment "URL is secret" is security through obscurity and the URL is in web server logs, browser history, and Referer headers.
- **[Critical]** `action=template` branch runs `subst $tmpl` on the raw `template` parameter â€” `[exec ...]` inside the template executes. Equivalent to the `tcl` branch with a different parameter name.
- **[Critical]** `action=diag` branch builds `"run_diagnostic $host $cmd"` and `eval`s it. Both variables are attacker-controlled and already subst-decoded; this is arbitrary Tcl execution via two separate injection points.
- **[Critical]** `action=plugin` calls `source "/etc/sysadmin/plugins/$cmd.tcl"`. `cmd=../../../tmp/payload` (with attacker writing to `/tmp/payload.tcl` via another branch or external means) sources arbitrary Tcl. Path traversal plus arbitrary code execution, no `catch`.
- **[Critical]** `exec ping -c 4 $host`, `exec traceroute $host`, `exec tail -n $count $path`, `exec grep -rn $pattern /var/log`, `exec psql ... -c $q`, `exec telnet $host $port`, `exec curl -s http://$host/health`, `exec /etc/init.d/$svc restart` â€” every one of these uses the single-string `exec` form against attacker-controlled values that have already been subst-processed. Any metacharacter (`;`, `|`, `&`, `` ` ``, `$()`) executes additional shell commands. `host=localhost;cat /etc/shadow` works.
- **[High]** `action=viewlog` and `action=tail` build paths as `"/var/log/$logfile"` with no traversal check. `logfile=../../etc/shadow` reads `/etc/shadow`. The `catch` in `viewlog` leaks the full resolved path back in the error message.
- **[High]** `action=fetch` is a server-side request forgery primitive â€” `curl -s http://$host/health` against an arbitrary `$host`. Reaches cloud metadata services (`169.254.169.254`), internal admin interfaces, and any reachable host the firewall permits to the CGI server.
- **[High]** `action=dbquery` builds SQL by interpolation: `"SELECT host, metric, value FROM metrics WHERE host='$filter'"`. SQL injection through `psql -c`, with the additional twist that the shell processes `$filter` before psql does, so `$()` and backticks execute first.
- **[High]** `action=viewlog` writes raw log contents into HTML with no escaping (`puts "$line<br>"`). Anyone who can inject a string into any log file under `/var/log/` (syslog, auth log, web access log) gets stored XSS executing in the admin's browser.
- **[Medium]** `action=dmesg` runs `regexp $pattern $lines` with a user-supplied pattern. Catastrophic backtracking patterns (`(a+)+$` against a long input) consume CPU until the request times out â€” denial of service per request, trivially repeatable.
- **[Medium]** No `catch` around `exec`, `open` (in `tail`, `search`, etc.), `source`, or `regexp`. Errors raise Tcl exceptions that terminate the script after `Content-Type` is sent but before the response body, leaving the client with a truncated page and the error in the web server log (where it may also leak filesystem paths).
- **[Medium]** `get_param` does no URL decoding itself, then `url_decode` is called on only four of the ten parameters. `logfile`, `svc`, `port`, `count`, `tmpl`, and `action` are used raw â€” including `tmpl` which goes straight to `subst`. Percent-encoded payloads bypass the (already broken) decoder when the branch reads the parameter directly.
- **[Medium]** All fifteen action checks are independent `if` blocks rather than `if/elseif` or `switch`. Currently only one matches per request because `action` holds a single value, but the structure invites bugs if anyone adds an action that mutates `$action` or shares parameter names.
- **[Low]** `package require Tcl 8.0` pins the interpreter to a version released in 1997. Modern Tcl 8.6+ provides `try`/`finally`, list-form `exec`, and `interp create -safe` improvements that this code needs.
- **[Low]** No `X-Content-Type-Options`, no charset on `Content-Type`, no CSRF token. Any logged-in admin visiting a malicious page triggers arbitrary actions via image tags pointing at this CGI.

## Modernization Path

This script cannot be incrementally hardened into a safe state. The dispatcher, the decoder, and every action branch are independently exploitable; fixing one leaves fourteen other RCE paths. The realistic options are retire-or-rewrite. Concrete fixes below assume rewrite; minimal fixes are marked where they meaningfully reduce blast radius before rewrite.

- **(Rewrite scope)** Replace the entire CGI with an authenticated admin tool behind a real auth layer â€” at minimum HTTP Basic over TLS with a non-default credential store, ideally an SSO-integrated service. The current "URL is secret" model must be abandoned before any other change matters.
- **(Rewrite scope)** Replace ad-hoc query parsing with `ncgi` (Tcllib): `package require ncgi; ncgi::parse; set host [ncgi::value host]`. `ncgi::value` returns the decoded value without invoking `subst`. Delete `url_decode` entirely.
- Replace `subst $str` in `url_decode` with proper percent-decoding only: `regsub -all {%([0-9A-Fa-f]{2})} $str {\\x\1} str; set str [subst -nocommands -novariables $str]` â€” or better, use `ncgi::decode`. `subst -nocommands -novariables` disables both `[...]` and `$var` expansion, leaving only backslash sequences.
- Replace every single-string `exec` with list-form, shell-bypassing `exec`: `exec -- ping -c 4 -- $host`. The `--` before the arguments stops option injection (`host=-R` against `ping`). For commands that accept hostnames, pass `$host` as a distinct argument â€” the shell is never invoked, so metacharacters become literal.
- Validate `host` against a hostname/IP regex before passing to `exec`: `if {![regexp {^[a-zA-Z0-9._-]+$} $host]} { error "invalid host" }`. Same for `svc` (allowlist of known services), `port` (numeric range), `count` (`[string is integer -strict $count] && $count > 0 && $count <= 1000`), `logfile` (allowlist of base names, no `/` or `..`).
- Replace `dbquery`'s `exec psql -c $q` with the Tcl `pgtcl` (or `tdbc::postgres`) binding using parameterized queries: `set stmt [$db prepare {SELECT host, metric, value FROM metrics WHERE host = $1}]; $stmt execute [list $filter]`. Removes both SQL and shell injection.
- Replace `viewlog`'s path construction with a fixed allowlist: `set allowed_logs {syslog auth.log dpkg.log}; if {$logfile ni $allowed_logs} { error "unknown log" }; set path "/var/log/$logfile"`. Then `open $path r` with a `catch` whose error message is logged server-side but reported to the client only as a generic failure.
- Replace raw log output into HTML with `htmlEscape`: write a `proc html_escape {s} { return [string map {& &amp; < &lt; > &gt; \" &quot; ' &#39;} $s] }` and pass every log line through it before `puts`. Eliminates stored XSS from log content.
- Delete the `tcl`, `diag`, `template`, and `plugin` branches outright. There is no safe form of `eval $user_input`, `subst $user_input`, or `source $user_path`. If operators genuinely need a plugin mechanism, implement it as: server-side allowlist of plugin names, `interp create -safe sandbox; $sandbox eval [list source /etc/sysadmin/plugins/$name.tcl]`, with explicit `interp expose` for the minimal command set the plugin needs.
- Replace `fetch`'s `curl` shell-out with the Tcl `http` package and an explicit hostname allowlist: `package require http; set allowed {monitor1.internal monitor2.internal}; if {$host ni $allowed} { error }; set tok [http::geturl "http://$host/health" -timeout 5000]`. Blocks SSRF to metadata services and arbitrary internal hosts.
- Replace `regexp $pattern` in `dmesg` with `string match` and a glob pattern, or compile and validate the regex length before use: `if {[string length $pattern] > 64} { error }; if {[catch {regexp -- $pattern $lines match} err]} { ... }`. Prevents ReDoS via pattern complexity bounds; full mitigation requires moving to a non-backtracking regex engine, which Tcl does not provide.
- Wrap every `exec`, `open`, `source`, and `socket` call in `catch`: `if {[catch {exec -- ping -c 4 -- $host} output]} { puts "Diagnostic failed"; log_error $output; return }`. Error details go to the server log, never to the client.
- Replace the chain of independent `if` blocks with `switch -- $action { ping {...} trace {...} default { error "unknown action" } }`. Eliminates the fall-through structure and forces an explicit allowlist of action names.
- Add `Content-Type: text/html; charset=utf-8`, `X-Content-Type-Options: nosniff`, `Content-Security-Policy: default-src 'none'`, and a CSRF token check on every action that mutates state (`restart`, `plugin`) before any modernized version of this script is exposed.
